OpenIntelligence

Local-first document intelligence for iPhone and iPad. Import private libraries, inspect cited answers, and use an Apple-native retrieval stack that prefers grounded evidence over generic chat behavior.

App Store Support Privacy Policy

Shipping App Plus Engine Lineage

OpenIntelligence is the public app surface for a broader document-intelligence stack, with the reusable engine boundary living behind the product.

View on GitHub

OpenIntelligence ingests PDFs, Office files, code, images, audio, and video into private libraries, then retrieves and answers from that material with citations, confidence checks, and deliberate abstention when the evidence is weak. Most ingestion, indexing, and retrieval stays local; when Apple-managed cloud capacity is used, it is only through platform-controlled paths rather than generic third-party AI hosting.

What Makes OpenIntelligence Different

Local-first by default

PDF parsing, OCR, library storage, FTS indexing, and retrieval are designed to stay on device unless the user explicitly crosses into an Apple-managed cloud path.

Broad document ingestion

PDFKit, Vision OCR, structured parsers, and media extraction paths handle digital documents, scanned pages, code, images, audio, and video.

Hybrid retrieval stack

SQLite FTS5, local vector indexing, query analysis, reranking, and context packing are used together instead of a single-vector-only search path.

Cited answers and abstention

The app is built to show evidence, drop unsupported claims, and refuse when the documents do not support a confident answer.

Library and container isolation

Private libraries, retrieval controls, and diagnostics make it possible to scope work and inspect how the answer path behaved.

App and engine split

The public product ships as an app, but the underlying document intelligence boundary is also treated as a reusable engine inside the broader OpenIntelligence line.

Technical Profile

Core APIs

  • UI: SwiftUI on iPhone and iPad
  • Parsing: PDFKit, Vision OCR, structured document processing
  • Models: Apple Foundation Models
  • Retrieval: SQLite FTS5, local vector indexing, reranking

Data boundary

  • Local: Libraries, normalized text, retrieval state, citations
  • Optional: Apple-managed cloud capacity only through supported platform paths
  • Not the product: generic chatbot behavior over arbitrary cloud context
  • Status: shipping public app plus broader engine lineage

How OpenIntelligence Runs

1

Import into private libraries

Files are assigned to local libraries and normalized through text extraction or OCR, depending on document quality.

2

Build lexical and vector indexes

The app preserves structure where possible, then stores lexical signals and vector representations for hybrid retrieval.

3

Retrieve and verify evidence

Query planning, reranking, and evidence packing are used to keep the answer path tied to source material rather than generic model guesswork.

4

Render answers with support or abstain

Answers are shown with citations and confidence cues, or the app abstains when support is not strong enough.
